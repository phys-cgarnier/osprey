API Reference#

Complete technical reference for all public APIs in the Osprey Framework. This is your authoritative source for classes, methods, functions, and their precise interfaces. See also the Developer Guides for learning-oriented content, then return here when you need specific implementation details.

🎯 API Learning Paths & Quick Navigation

Essential API Learning Path

  1. Base Components - BaseCapability and decorators

  2. State and Context Management - AgentState and ContextManager

  3. Gateway - Entry point architecture

  4. Human Approval - Safety and oversight

  5. Classification System - Robust error handling

Core API Patterns:

  • osprey.base.BaseCapability with @capability_node decorator

  • osprey.state.AgentState for LangGraph-native state management

  • osprey.context.ContextManager for type-safe data exchange

  • osprey.registry.RegistryManager for component discovery

Advanced API Workflows

  • Orchestration - ExecutionPlan creation and coordination

  • Python Execution - Secure code execution APIs

  • Data Management - Unified data source integration

  • Container Management - Deployment and service management

  • Recovery Coordination - Recovery strategies and coordination

Production-Ready APIs:

  • osprey.approval.ApprovalManager - LangGraph-native approval workflows

  • osprey.data_management.DataSourceManager - Provider discovery and retrieval

  • osprey.deployment.container_manager - Service orchestration and deployment

  • osprey.models.get_chat_completion() - Multi-provider LLM management

  • Configuration System - Environment and settings management

  • Framework Utilities - Logging, streaming, and observability

Implementation Shortcuts

Build a new capability → BaseCapability + capability_node()

Add approval workflows → ApprovalManager + Human Approval

Execute Python code safely → PythonExecutorService

Store user context → ContextManager + State and Context Management

Deploy with containers → container_manager + Container Management

Handle complex data flows → DataSourceManager

Integrate external data → DataSourceProvider + provider patterns

Manage LLM models → get_chat_completion()

Configure logging and streaming → get_logger() + ComponentLogger
